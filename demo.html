场景二：龙虾不回应了？让 Kiro 排查复活
OpenClaw Gateway 挂掉时，用一句自然语言让 Kiro 完成从诊断到修复的全过程。
💥
Config 配置错误
openclaw.json 改错了导致 Gateway 无法启动，Kiro 自动检查配置文件、定位错误字段、修复并重启
🔋
进程意外崩溃
Gateway 进程挂了，Kiro 检测进程状态、查看错误日志、重启服务，整个过程 30 秒内完成
🌐
依赖包版本冲突
npm 依赖缺失导致启动失败，Kiro 读取错误堆栈、执行 npm install、重新启动服务
"我的龙虾不回应了，帮我排查一下"
已检查：Gateway 进程未运行 · 日志显示 Config invalid（memorySearch.apiKey 字段不存在）· 已修复配置 · Gateway 重启成功 · OpenClaw 正常响应

Nova Sonic — 语音对话 Demo
Amazon Bedrock Nova Sonic · 端到端实时语音交互
真正的端到端语音 AI——不是语音转文字再问 AI 再转语音的三段式架构，而是 Nova Sonic 直接理解语音、思考、语音回复。支持打断，有情绪感知，平均延迟 <1秒。
🎙️ 点击体验 Nova Sonic 语音对话
# Nova Sonic 技术架构
用户说话 → Nova Sonic 实时理解 → 思考 + 生成语音 → 用户听到回复
# 传统三段式架构（对比）
用户说话 → STT → LLM → TTS → 用户听到回复 （平均延迟 3-4s）
# Nova Sonic 优势
✓ 端到端，延迟 <1s
✓ 支持打断（barge-in）
✓ 情绪感知，语气自然
