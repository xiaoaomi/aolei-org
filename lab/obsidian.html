Obsidian 笔记云同步

用 AWS S3 做 Obsidian 的同步后端——比 iCloud 便宜，比官方同步灵活，数据在自己手里。
手机、Mac、PC 三端同步，随时随地打开都是最新的。

同步架构

核心方案很简单：在每台设备上安装 Remotely Save 插件，指向同一个 S3 Bucket。 同步时插件把变更的文件上传到 S3，其他设备下次同步时拉取更新。 没有中间商，没有月租费，数据加密后传输。

同步架构
📱 iPhone · 💻 Mac · 🖥 PC
→
🔌 Remotely Save 插件
→
☁️ AWS S3
ap-southeast-1 新加坡
↔
✓ 多端同步
🔒 端对端加密：文件在设备上加密后上传，S3 存储的是密文，即使 Bucket 泄露也无法读取内容

为什么选 S3，而不是 iCloud

对比维度 iCloud / 官方同步 AWS S3 方案
价格 官方同步 $8/月；iCloud 50GB $0.9/月但和系统绑定 约 $0.02/月（1GB 笔记库）
数据主权 数据在 Apple/Obsidian 服务器上 你的 AWS 账号，你的 Bucket，你的密钥
加密 iCloud 有加密，但 Apple 持有密钥 端对端加密，只有你有密钥
跨平台 iCloud 在 Android/Windows 上体验差 iOS / Android / macOS / Windows 全支持
配置难度 零配置，开箱即用 需要创建 S3 Bucket + IAM 用户（约 20 分钟）
稳定性 依赖服务商 AWS S3 99.999999999% 持久性

怎么配置

1
创建 S3 Bucket
AWS 控制台 → S3 → 创建 Bucket，建议选新加坡（ap-southeast-1，国内延迟最低）。 关闭「公开访问」，开启版本控制（防误删）。
2
创建 IAM 用户
创建一个只有 S3 读写权限的 IAM 用户，生成 Access Key + Secret Key。 不要用 Root 账号，权限最小化原则。
权限：s3:GetObject, s3:PutObject, s3:DeleteObject, s3:ListBucket
3
安装 Remotely Save 插件
Obsidian → 社区插件 → 搜索「Remotely Save」安装启用。 在插件设置里选 Amazon S3，填入 Bucket 名、区域、Access Key。
4
开启端对端加密（可选但推荐）
插件设置 → 加密 → 开启，设置一个密码短语。 加密后所有文件以密文形式存储在 S3，AWS 也看不到你的内容。
注意：所有设备用同一个密码短语，否则无法互相解密
5
其他设备重复步骤 3-4
手机、iPad、其他电脑，安装 Obsidian + Remotely Save，填写相同的配置， 首次同步时选择「使用远端版本」即可完成初始化。
